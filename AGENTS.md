# AGENTS.md

Compact guide for AI agents working in this repo. For full architecture details, see `CLAUDE.md`.

## Commands

```bash
npm run dev                    # Dev server at localhost:3000
npm run build                  # next build + copies static into standalone (scripts/sync-standalone-assets.mjs)
npm run lint / npm run lint:fix
npm run format / npm run format:check   # Prettier on src/**/*
npm run type-check             # tsc --noEmit
npm test -- --run              # Single test run (default npm test is watch mode)
npx vitest run src/lib/errors.test.ts   # Single file
npm run test:coverage
```

CI runs in order: `lint → type-check → test --run → build` (see `.github/workflows/docker.yml`).

CI sets dummy env vars (`NEXT_PUBLIC_GITHUB_CLIENT_ID=test`, `GITHUB_CLIENT_SECRET=test`) so build succeeds without real secrets. Locally, copy `.env.example` → `.env.local` with real values.

## Conventions

- **Prettier**: double quotes, semicolons, 100-char width, trailing commas es5. Don't default to single quotes.
- **Import alias**: `@/*` maps to `src/*`. Always use `@/lib/...` not relative `../../lib/...`.
- **Commit format**: `type(scope): description` (e.g. `feat(sync): improve URL metadata fallback`).
- **Component files**: PascalCase. Utility/hook files: kebab-case.

## Non-obvious structure

- **Two `data/sites.json`**: `src/data/sites.json` is the default seed fallback (committed). `data/sites.json` is runtime user data synced with GitHub (gitignored). Don't confuse them.
- **Tailwind v4 — no `tailwind.config.js`**. Theme is CSS-only via `@theme inline` in `src/app/globals.css`. Use CSS custom properties (`var(--primary-500)`) for colors, not Tailwind color classes.
- **Middleware at `src/middleware.ts`**, not root-level. Sets CSP, HSTS, security headers. CSP is built dynamically by `buildContentSecurityPolicy()` from `src/lib/runtime-policies.ts`.
- **`server.js` in project root** is generated by `next build` with `output: "standalone"`. Don't edit it — it's a build artifact.
- **`NEXT_PUBLIC_GITHUB_CLIENT_ID`** is served at runtime via `/api/runtime-config` (see `src/lib/runtime-public-config.ts`). Docker images are built once and configured at container startup.

## State & data flow

Single React Context (`SitesContext` in `src/contexts/SitesContext.tsx`) — no external state library. All CRUD, sync state, and guest mode live here.

```
User action → localStorage (instant) → UI update → 3s debounce → /api/github/data → GitHub API
```

Sync uses fingerprint-based conflict detection (rejects silent overwrite when both sides changed). See `src/lib/storage/sync-manager.ts`.

## Styling

- shadcn/ui pattern: `cn()` utility (clsx + tailwind-merge) in `src/lib/utils.ts`
- Component classes are defined in `globals.css` (`@layer components`): `.card`, `.site-card`, `.input`, `.empty-state`, etc. — use these, don't recreate them.
- Dark mode via `prefers-color-scheme` media query, not a class toggle.

## Tests

- Vitest with jsdom, globals enabled. Setup in `src/test/setup.ts` mocks `localStorage` and `sessionStorage`.
- Tests colocated as `*.test.ts` / `*.test.tsx` near source.
- Coverage excludes `src/test/`, `node_modules/`, `*.d.ts`, config files, and `**/mockData`.

## Auth

GitHub OAuth → HttpOnly cookie → server-side token extraction. Frontend never sees the raw token. API routes in `src/app/api/auth/` handle the flow. Guest mode is read-only.

## Deployment

Docker multi-stage build (`Dockerfile`). The `npm run build` script runs `scripts/sync-standalone-assets.mjs` after `next build` to copy `.next/static` and `public` into `.next/standalone` — without this, standalone serves 404s for hashed chunks. Standalone server requires `HOSTNAME=0.0.0.0` env var or it binds to wrong address (502). CI deploys via SSH to production server.
